üß† RAG Strategy

What is RAG? Retrieval-Augmented Generation combines retrieval systems with generative AI. Instead of relying solely on LLM training data, we retrieve relevant context from our knowledge base.

Why RAG for Note Ninjas?

  • ‚úÖ Accuracy: Grounds responses in verified medical knowledge
  • ‚úÖ Consistency: Standardized exercise recommendations
  • ‚úÖ Traceability: Each suggestion links to source material
  • ‚úÖ Updateable: Easy knowledge base updates without retraining
  • ‚úÖ Cost-Effective: Reduces token usage vs. fine-tuning

üìù Feedback System

Granular Feedback Collection

We collect feedback at 4 levels:

  • ‚úÖ Exercise Level: Overall exercise recommendation
  • ‚úÖ Cue Level: Each individual exercise cue
  • ‚úÖ Documentation Level: Each documentation example
  • ‚úÖ CPT Code Level: Each billing code suggestion

Enhanced Payload Structure

Feedback now includes:

  • Full case context (patient condition, desired outcome)
  • Exercise hierarchy (name, description)
  • Item content (actual cue/doc/CPT text)
  • Timestamp and rating

Continuous Learning Loop

User submits feedback ‚Üí Stored with context ‚Üí Weekly analysis ‚Üí Knowledge base updated ‚Üí RAG retrieves improved content ‚Üí Better recommendations
